Luogu Submitting Crawler文件和目录

Luogu Submitting Crawler会生成一些目录和文件，以下是它们的信息。

这些目录和文件仅会在Luogu Submitting Crawler同级目录下生成。

Note

GNU General Public License许可证文件和源代码目录不在本章节讨论的范围内。附属组件和用户文档的相关信息已在上文中做出说明，同样不纳入讨论。

Warning

不要人为对本软件目录中的任何文件作任何更改，否则可能导致本软件不按预期运行。

本章节中的”目录“可作为”文件夹“理解。

目录

Note

在本章节中，_uid表示用户的luogu UID，pid表示luogu的题号。它们相当于两个变量。

Code：结构为code\_uid\*.*，分_uid存储所爬取的源代码。源代码根据不同的语言保存为不同的文件类型。

Record：结构为record\_uid\*.txt，分_uid存储存储代码有关参数。其中，pid.txt存储某一题的所有代码及其文件参数，pid_best.txt存储某一题最优代码参数。这两者除在软件爬取过程中一般不单独存在。否则表明软件工作可能存在问题。该目录下还可能存在last.txt，这记录了上一次爬取的最新运行号。这个文件为“增量爬取”提供了参照。

Problem：结构为problem\pid.txt，存储每一题的题目名称。由软件自动爬取得来，用户可在之后自行更改。

文件

tmp.txt，由本软件本体生成，存储一些临时数据或一些无法直接读取的内容。

setting.txt，由本软件本体生成，存储用户的设置信息。

log.txt，由本软件本体生成，存储本次软件运行时输出的错误数据（stderr）。sysGetor虽然会在软件启动时向其写入内容，但进程结束后内容即会被删除。

logArchive.txt，由本软件本体生成，存储之前的错误数据。

cookie.txt，由本软件本体生成，存储用户保存的Cookie（已进行加密）。当软件启动时发现Cookie失效，用户更新或删除本地Cookie时，该文件即被删除。

telemetry.txt，由本软件本体生成，存储需要telemetryAssist遥测的文件。

sysinfo.txt，由附属组件sysGetor和localFetcher生成，存储用户的系统数据。包括smBIOS UUID、MachineGUID、计算机名和用户名、地区信息。

updateTmp.txt，由附属组件updateChecker生成，存储更新检查时产生的临时数据或无法直接读取的内容。updateChecker启动时会将旧的文件删除。

update.txt，由附属组件updateChecker生成，存储获取到的最新版本号。updateChecker启动时会将旧的文件删除。

telemetryTmp.txt，由附属组件telemetryAssist生成，存储遥测时产生的临时数据或无法直接读取的内容。

telemetryStatus.txt，由附属组件telemetryAssist生成，存储当前其的状态。

telemetryTime.txt，由附属组件telemetryAssist生成，存储上一次成功遥测的时间和耗时。telemetryAssist启动时会将旧的文件删除。

Warning

要删除所有的目录和文件，可使用”还原“功能。数据删除后无法恢复，请谨慎操作。

设置Luogu Submitting Crawler并开始爬取

Important

本章节（设置Luogu Submitting Crawler并开始爬取）与下一章节（提取和管理已经爬取的代码）的内容是Luogu Submitting Crawler的流程使用指南，你可在这些部分学习如何使用Luogu Submitting Crawler。

本章节阐述了如何对Luogu Submitting Crawler进行设置，以做好爬取准备。

键入Cookie

Luogu Submitting Crawler需要你的Cookie才能爬取代码。因此需要你从浏览器获取Cookie后键入Luogu Submitting Crawler。我们需要_uid和__client_id。

在Luogu Submitting Crawler中，一组合法Cookie相当于一个用户。如果没有Cookie，你将无法使用其内的大部分功能。除了题目名称由所有用户共享，代码与代码参数均按照用户分类存放，且你无法对其他用户的代码和参数执行操作。区别用户的依据是_uid。

转到”管理Cookie-键入Cookie“来管理你的Cookie。

Note

我们将严格保护你的Cookie，不会向Internet发送你的Cookie。若你选择将Cookie保存到本地，我们将对__client_id基于你计算机的smBIOS UUID和MachineGUID进行加密。smBIOS UUID可能在不同计算机上重复，但MachineGUID是唯一的。若将加密的Cookie保存文件（cookie.txt）转移到另一台计算机上，则将无法解密。当然，我们也不会共享你的加密Cookie保存文件。

Caution

不要将Cookie发送给其他任何人。否则你的账号极易被盗用。

在获取Cookie前，你必须在近30天内登录过luogu.com.cn。获取Cookie的方法请自行查找，本文档不作说明。

Cookie验证成功后，你可以选择将其加密保存到本地。此时你将无需每次启动时重新键入Cookie。Cookie的有效期一般为30天。本软件将在每次启动时检查一次保存在本地的Cookie。若发现其失效，则会将保存在本地的Cookie文件删除。此时你可能需要重新登录，然后键入新的Cookie。

Tip

如果你认为你键入了正确的Cookie，但仍无法通过验证，请检查你的curl版本。如果不是最新版，请自行下载最新版本（注意处理器架构和位数）并将其二进制文件（包括其同级目录下的文件）放置在Luogu Submitting Crawler二进制文件同目录。然后再试一次。若仍不成功，请上报GitHub的Issue板块。并注明你的系统版本（类似于Windows 11 24H2；10.0.26100.4652）以便我们查找问题。

设定保存评判依据、保存文件名和网络请求参数

本章节阐述了保存代码时的评判机制以及代码文件名有关事项。

保存评判依据

Note

如果你启用了“保存所有AC代码”功能，则对于保存代码而言“评判依据”规则将失效。但本软件仍将根据此规则保存和更新最优代码参数。

Luogu Submitting Crawler可根据你设定的规则比较代码不同维度参数从而保存“最优”代码。本软件支持比较“时间，内存，代码长度，提交时间”四个维度。其中其三个维度越小越优，最后一个维度新/旧均可。你可调整这四个维度的比较顺序。当两个代码在某个维度上相同时，Luogu Submitting Crawler会继续比较下一个维度直至出现差异。

保存文件名

Luogu Submitting Crawler会自动按你的设置为爬取的代码命名。每一个名称的结构如下：pid probName submitTime rid.exName。其顺序无法修改。

其中，pid为该代码对应题目的题号（此项无法取消），probName为该代码对应题目的名称，submitTime为该代码的提交时间，rid为该代码的运行号，exName为该代码的扩展名（由代码的语言对应）。

Note

对于exName项，Luogu Submitting Crawler可自动为Pascal，C，C++，Python，Java，Rust，Go，Haskell，PHP语言设置其对应的扩展名，"提交答案"的扩展名为ans，其他语言一律保存为txt。

Caution

由于Windows的文件系统不支持某些字符，因此若原题目名称含有这些字符，则软件会将题目名称设为Illegal_File_Name。在手动修改题目名称时，也请避免这些不支持的字符，否则当这些题目名称被作为文件名时将出现错误。此时可能需要重新爬取所有代码。

当“保存所有AC代码”启用时，rid项必须被选择，否则可能导致文件重名。当你打开“保存所有AC代码”或取消选择rid项时，另外一个选项将会自动切换。

网络请求参数

当Luogu Submitting Crawler使用curl执行网络请求时，其会附带一些参数。你可根据需要进行调整。

请求持续时间

该值限制了网络请求的持续时间。如果请求超过该时间限制，则请求将停止并返回错误。

相邻请求间隔

该值限制了相邻两个网络请求的间隔时间，以使得网络请求不至于过快。

Tip

对于“请求持续时间”：如果你的计算机网络连接良好，速度较快，则可适当降低该值。若你计算机的网络速度不佳，则建议适当延长该值。

对于“相邻请求间隔”：如果你的计算机性能良好，则建议增加该值；否则可适当降低该值。

你也可以保持默认设置，一般不会有太大影响。

选择爬取模式

本章节阐述了不同爬取模式的差异，以及“夜间计划爬取”的特性。

在以上章节所提及的设置均完成后，转到“开始爬取”，然后选取不同的爬取模式。

一般爬取

该模式在爬取前会清除本luogu账户下所有爬取的代码及代码参数，并重新爬取所有的代码。

该模式的爬取记录将为下一次的增量爬取提供参照。

单题爬取

该模式会在爬取前删除指定题目在本地保存的文件名，以及本luogu账户指定题目爬取的所有代码及代码参数，并重新爬取指定题目名称及其所有代码。

该模式的爬取记录无法用于增量爬取。

增量爬取

该模式不会删除任何文件，会爬取代码直至上次已经爬取的部分。该模式需要上一次的记录文件作为参照。

该模式的爬取记录将为下一次的增量爬取提供参照。

Note

如果增量爬取不可用，则其将不会显示。

Tip

如果你是第一次使用Luogu Submitting Crawler或之前使用过但旧版本参数文件与新版本不兼容，则建议使用一般爬取。

如果某一道题进行了重测或题目名称进行了更新，或者你只需要某一题的代码，建议使用单题爬取。

如果之前已爬取了部分代码，增量爬取可用且你希望继续更新你本地的代码，建议使用此模式。该模式可降低网络请求数量和重复爬取的次数，更好减轻服务器的负担并节省能源。

爬取模式设置完成后，你可决定是否启用夜间计划爬取。

夜间计划爬取

夜间计划爬取设置后，Luogu Submitting Crawler会在23:00开始爬取。当爬取完成后，如果夜间计划爬取没有退出，则Luogu Submitting Crawler将自动关闭计算机。关闭计算机将有60秒的超时，在这段时间内你可以取消关机。

如果夜间计划爬取进入等待阶段时你希望立即开始爬取，请按下esc键，此时夜间计划爬取将退出并立即开始爬取。如果你在夜间计划爬取自动开始后希望退出该模式，也请按esc键中止爬取，然后重新爬取即可。退出夜间计划爬取不会影响代码爬取。若代码爬取时有任何中断，如果此时夜间计划爬取处于开启状态，也将自动退出并暂停爬取进程以便用户确认。

若你希望使用该功能，请确认计算机在夜间能够正常运行且网络活动不会中断。

Tip

该模式可以有效避开服务器流量高峰，同时节省能源开支。

Warning

该模式在夜间处于完全“无人监管下运行”状态。如果你的计算机不稳定或你决定本模式存在问题（请上报Issue），请勿使用以免造成不良后果。

开始爬取

以上设置均完成后，你可开始爬取。

Note

此过程可能需要较长时间，具体时间消耗依你计算机的网络和处理器性能，爬取量的大小，服务器流量等因素而定。

在爬取过程中，若你希望暂停或退出爬取，请按esc键。如果爬取中遇到错误，则爬取也将自动暂停。你可以选择重试，也可以退出爬取。

提取和管理已经爬取的代码

本章节阐述了如何提取和管理已经爬取到本地的代码。

Note

要执行本章节的操作，你必须具有一个合法的Cookie。你不能对其他luogu账户的代码/参数文件执行操作。

Warning

请仔细阅读本章节说明，任何批量操作的错误都有可能导致需要重新爬取所有代码。请谨慎操作。

提取已经爬取到本地的代码

转到“管理,提取本地代码-提取代码到指定文件夹”以提取代码。

Luogu Submitting Crawler会将你luogu账户中所有的已爬取代码拷贝到你指定的路径下的文件夹code_uid文件夹下（uid为你的_uid）。

Caution

你指定的路径应该是绝对路径，且不应包含任何非ASCLL字符或不可显示的ASCLL字符，否则极易导致提取失败。

Tip

我们建议在执行下文所提及的操作时，先提取出代码作为备份（并将你luogu账户的参数文件复制一份，注意不要做任何更改）再执行操作。当不慎出现错误时，你可以立即还原。这将有效避免重新爬取所有代码的需要。

手动修改指定题目名称

你可手动更改某一题的题目名称。然后，Luogu Submitting Crawler会依据该新名称为你luogu账户下的所有本题代码依据当前的文件名设置进行重命名。当题目的名称被更新或原题目名称中具有Windows路径不支持的字符时，该功能十分有用。

Note

只有你luogu账户下有该题目的Accepted代码，你才可以修改这一题的文件名。

Warning

虽然你修改某题的名称后，Luogu Submitting Crawler只会对你luogu账户下的指定题目进行重命名，但当其他luogu账户用户使用本计算机上的Luogu Submitting Crawler时，他们也会使用这一被修改的题目名称。文件名数据是所有luogu账户共享的。只要他们爬取了代码，或选择将他们的代码按照格式重命名，则这一被修改的题目名称也将体现在他们的代码文件名上。

Caution

在手动修改题目名称时，你应该避免键入Windows路径不支持的字符，否则当该名称被作为文件名的一部分时将导致错误。并且这一错误可能影响其他luogu账户用户。

将代码按当前设定的格式重命名

代码爬取完成后，你可更改已爬取代码的文件名格式。代码的文件名格式以当前设定的文件名格式为准。

Caution

如果之前开启过保存所有AC代码功能，则必须开启“运行号”一项。该值对于每个提交记录是唯一的，否则可能导致代码重名。

Note

本操作可能需要较长时间，具体时间消耗取决于你计算机的性能和代码数量。

删除爬取的代码和参数文件

关于遥测及其隐私

遥测的目的是让我们了解用户在使用过程中可能出现的错误并针对性地改进软件。我们尊重并保护你的隐私，因此我们确保在遥测时落实以下几点：

  • 不会遥测任何敏感或秘密数据（例如你的Cookie，用户名等）。

  • 使用不与你和你的计算机关联的数据（即Telemetry ID，是由你计算机的smBIOS UUID和MachineGUID组合后求得的MD5码。MD5码是一种哈希算法，无法通过计算得知你计算机的这两个ID）来区分遥测数据的发送计算机。

  • 减少遥测数据头的信息。遥测将仅发送你计算机的系统版本（例如：Windows 11 Pro 24H2）、Telemetry ID和地区信息（例如CN，zh-CN，2052）。

  • 使用SMTP over SSL发送遥测数据。

  • 不会向任何人共享收到的遥测数据。

  • 遥测数据仅在中国大陆存储和使用。

这些措施将有效保证遥测过程中的数据和隐私安全。
